# Connect an Agent to Slack

## Summary

An Agentforce Employee agent can be connected to Slack by connecting the Salesforce org and Slack, adding the Slack connection to the agent, granting user access, activating the agent, and installing it in the appropriate Slack workspaces.

## Key Concepts

- Agentforce Employee agent
- Slack connection
- Salesforce and Slack connection
- Agentforce Builder
- Agent activation
- User access
- Slack workspace
- Salesforce permissions

## Source Content

First connect Salesforce and Slack. In Agentforce Builder, add the Slack connection to the Agentforce Employee agent.

For the legacy builder, install the Slack external client app and add Slack as an external app in the agent's Messaging connection.

Grant users access to the agent and activate it. When users interact with the agent in Slack, the agent respects their Salesforce licenses, permissions, and other access controls.

For a Pro or Business+ Slack subscription, a Slack administrator opens Manage Agentforce, reviews the agent's permissions, and installs the agent.

For an Enterprise Slack subscription, use the Slack organization settings to open Salesforce and Agentforce, review the agent, select one or more workspaces, and add the agent to those workspaces.
