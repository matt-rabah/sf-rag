# Canvas and Script Views in Agentforce Builder

## Summary

The new Agentforce Builder is organized into components including the Explorer, Canvas, Preview, and a Canvas/Script toggle.

Canvas view is a low-code, document-style editor, and Script view gives pro-code control, and edits stay in sync across both views because the builder writes Agent Script in the background.

## Source Content

The new Agentforce Builder is organized into several main components. The Explorer lets you view and navigate all the elements and assets of your agent, including the subagents and their actions, in a single place. The Canvas is a document-style editor you use to modify your agent or view its details. The Preview lets you test your agent by inputting utterances, and when previewing a conversation you can see a summary of your agent's reasoning and behavior for each message. The Canvas/Script toggle switches between the natural-language based Canvas and the code-based Script views. The Agentforce Assistant helps you build and debug your agent. Save saves the current version of your agent, and Commit Version saves a snapshot of the current version of your agent's metadata, which locks the version so you can activate the agent or create a new draft.

When you open or create an asset in Agentforce Builder, it opens in Canvas view by default. This low-code view simplifies agent configuration by summarizing Agent Script into easily understandable blocks. The canvas also gives you easy access to common tasks. You use inline actions as shortcuts for adding deterministic logic, such as if/else conditionals, and there's a resource picker for subagents, actions, and variables. For pro-code control over your agent, you switch to Script view to use the scripting language.

Whether you build your agent in Canvas or Script view, your agent's configuration and any edits you make are consistent across both views. For example, if you use Canvas view to modify your agent using natural language, the builder writes Agent Script for you in the background so the views stay in sync.
