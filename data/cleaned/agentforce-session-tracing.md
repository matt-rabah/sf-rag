# Agentforce Session Tracing

## Summary

Agentforce session tracing captures detailed information about agent sessions so admins and builders can observe, monitor, investigate, and troubleshoot agent behavior.

## Source Content

Agentforce session tracing captures details on agent interactions with users from start to finish.

Session tracing combines data from different sources under a session ID.

The session trace log provides detailed information about what happened during agent sessions.

Session tracing can include turn-by-turn interactions, reasoning engine executions, actions, prompt and gateway inputs and outputs, error messages, and final responses.

Session trace data can be used to observe, monitor, investigate, and troubleshoot agent events, incidents, and behavior patterns.

Session trace data is accessible through Data Model Objects.
