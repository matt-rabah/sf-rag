# Deploy the Agent to Your Experience Cloud Site

## Summary

Deploying an Agentforce Service agent to an Experience Cloud site adds the components needed for site visitors to use the agent through Enhanced Chat.

## Key Concepts

- Agentforce Service agent
- Experience Cloud
- Digital Experiences
- Enhanced Chat
- Omni-Channel
- Agent Readiness Checklist
- Experience Builder
- Digital Engagement

## Source Content

Before deployment, create an Agentforce Service agent from a template and create an Experience Cloud site. Provision Digital Engagement in the Salesforce org, and confirm that the org uses Omni-Channel or Enhanced Chat.

From Setup, open Agentforce Agents and select the agent to deploy. On the Agent Details page, open the agent in Agentforce Builder.

In the Agent Readiness Checklist, select Deploy Your Agent to an Experience Cloud Site and choose Go to Sites. Select the target Experience Cloud site, click Deploy, verify that the prerequisites are complete, and confirm the deployment.

The deployment adds the necessary components to the selected site page. In Experience Builder, verify that the Enhanced Chat component is present. If it is missing, configure Enhanced Chat in Setup.

The chat panel does not open automatically in Experience Builder preview mode. A user must select the chat icon to open it.

This source supports the AI Agents objective about connecting an agent to a digital experience. It identifies the Experience Cloud prerequisites, the builder deployment path, and the Enhanced Chat component required on the site.
